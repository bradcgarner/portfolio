I am a full-stack web developer.

I revel at the challenge of making complex solutions appear simple. I enjoy the depth and breadth of development problem-solving: specifications, user interactions, modeling, wireframing, programming, documentation, and testing. I’m drawn to the web’s infinite scalability and interconnectedness that fits in your pocket.

My skills and interests span the client- and server-sides. I enjoy writing API documentation just as much as I enjoy building React apps.

How I got into web development

I discovered a love of programming in college, frequently finding myself enthusiastically writing Pascal (yeah, this was the early 90s) programs to read ASCII files of “tank movements” at 3 am. This led to a minor in computer science.

I then happily pursued my first passion, landscape architecture, for two decades, incorporating programming in many useful ways, from building construction cost calculators to full ERPs.

Eventually I was nearly full-time on FileMaker WebDirect apps. Feeling my next calling, I started taking night classes in HTML, CSS, JS, and PHP, I then enrolled in Thinkful.

The web is the first mass media that allows two-way communication.

PROJECTS

Code To Go

Rationale

There are already so many great resources to learn to code, but a niche remains for learning on the go.

People check Facebook or Twitter on their phone, if only to fill a few idle minutes. Why not convert that time into mastering a language or framework!?

Let's say you have 10 minutes to spare waiting for a bus, between meetings, etc. You could open the app, answer multiple questions, get immediate feedback, and leave with at least 1 or 2 concepts in your head. You'll probably continue to think about those. That kind of intermittent learning can be very powerful, especially if put to action soon.

Key Features

Concealed Answers: User cannot see the answers with developer tools or otherwise.

Instant Feedback: You don't have to get to the end of a quiz to see how you are doing.

Stop-And-Start: You can do as much or little as you like, close the app, and return exactly where you left off.

Links to Resources: This is a quiz, not a tutorial. Link to sites where you can learn.

Challenges

Most of the key features (concealed answers, instant feedback, ability to stop and start) were also primary challenges.

Instant feedback while concealing answers requires scoring on the server. This round-trip with no latency perception is most challenging on 3G. Migration to an iOS or Android app could solve this problem.

Solving the instant feedback problem also solves most of the data persistence issues: Since the server is pinged with each question answered, the user can stop at any time, and when reloading the app, can pick back up at the last unanswered question.

Upcoming

Allow review of the entire quiz upon completion.

Allowing user to see the key (an track to disallow re-taking for credit after that).

Creating rankings/badges, utilizing user's scores and history.

Content: I have started a companion app for daily collection of the questions in my head as I learn, troubleshoot, and debug. I will also begin soliciting content from others as part of the app.

Look for this on codetogo.io soon.

JavaScript / ES6

JavaScript is the backbone of this app. ES6 features used include default parameters, template literals, destructuring, arrow functions, promises, let & const for clean, stable code.

Node

Lightweight, scalable server-side language ideal for this type of application.

Express

Server-side MVC organization.

RESTful API

All endpoints use RESTful principles. Endpoints include authentication, users, quizzes, questions and answers.

Passport Authentication

All protected endpoints utilize Passport middleware.

JavaScript Web Tokens

Upon successful login, a JWT is issued to the user, which is stored in the Redux store and in the browser's local storage for access to all protected endpoints.

Mongo noSQL Database

This app has a simple data model with minimal relational needs, so MongoDB fit the bill for ease of use.

mLab

All data is hosted at mLab. There are 4 collections: quizzes, questions, users, and choices (users' answers to questions).

Mocha

Server-side test suite uses the Mocha framework and Chai assertion library.

Chai

Server-side test suite uses the Mocha framework and Chai assertion library.

Travis CI

Travis CI was used to allow continuous testing during development.

HTLM5

New HTML5 features used include semantic elements, new form features, and placeholder text.

React

The client-side framework is React, used for its grace and speed of DOM manipulation.

Redux

This app extended React via Redux for superior state management. This app might seem very simple, but there's a ton going on in the state, and Redux is a great tool for that. Redux-form and Redux-thunk are used.

Jest

Client-side (React) tests were written in Jest and Enzyme, using the Chai assertion library.

Enzyme

Client-side (React) tests were written in Jest and Enzyme, using the Chai assertion library.

CSS3

The primary CSS3 feature used in this app is Flexbox. I love Flexbox! The entire app uses Flexbox. I use a few other new CSS3 featues on this page (my portfolio page).

SASS

I wrote the CSS for this app in SASS for speed of evaluating color schemes (using variables) and to organize code, being careful not to nest too much.

Github

The project's client-side repo and server-side repo are here and here.
