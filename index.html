Philosphy

The web was built to share freely, collaborate openly, and interact in real time. The web should allow us to know more, interact better, and be healthier than any other time in human history.

Though the web is exciting and fun, it is our most powerful communication medium, and something that comes with great responsibility. We web developers should guard again misuses such as weaponization of information, development of bad habits, and division.

Technology is just a means to an end. One of the best software skills is people skills: empathizing with users and being highly skilled at using technology to meet users' needs.

graphing

The monitoring project (above) requires graphing and data visualization of a wide range of tests and experiments. We at Green Roof Diagnostics never know what we'll need to graph.

I like chart.js and react-chart-js, but they require more configuration than is reasonable for a dynamic project. So I built a preprocessor to manage this for me.

The preprocessor requires the data to be formatted as an array of objects, and each object key should be pre-defined. This aligns perfectly with a normal SQL database setup, so that's no problem.

The preprocessor creates selectors for the data, manages Y-axes, allows X-axis shifts, allows users to save pre-set configurations, and more.

This NPM module is not publicly documented yet, but it could be a great asset for similarly complex projects.

Use this here or watch it here.

Environmental Monitoring

Overview

StormWatch: Internal application for collecting, managing, and analyzing data from an IoT sensor network, and R&D trials. Data sources include outdoor green roof platforms, a weather station, and a rain simulator.

Application allows for fast, consistent, accurate testing and monitoring of multiple trials.

One of the core requirements of this application is data integrity. We've got nearly 100 sensors, most of which need to have near 100% up-time. The app includes extensive monitoring of data consistency, varying alarm levels, and the ability to patch inconsistent data when redundant data sources are available.

Overview

This app connects charitable organizations and volunteers.

Organizations and individuals can post opportunities to volunteer, or even post offers to help others. The application handles opportunity signup, and allows the opportunity's host to accept or reject volunteers.

Process

This was my final project at Thinkful, built as part of a 3-person team. My primary responsibility was Redux, but I also took the lead in data modeling and API documentation.

Our team began this as our first test driven development (TDD) project, a great learning experience. My two key takeaways were that TDD can be a very powerful and effective way to write clean code, and that TDD is only very effective if the team is skilled at writing the necessary tests. Due to time constraints and lack of familiarity with some test protocols, we switched from TDD to testing while developing. It's a start...

Challenges

Consistency: With three people working rapidly on different parts of a project, consistency was important. API documentation and early agreement on naming conventions was critical.

React Widget Date/Time Picker: This one little widget got a disproportionate amount of my time. I ended up writing a lot of custom JS to get it to perform as I expected out of the box. When I investigate a little further, I'll blog about it and post a link here.

Learning TDD: Our team had difficulty getting to MVP quickly while starting the project as TDD.

Upcoming

There is a lot of potential for improvement, and I will probably use this project as a sandbox for learning and implementing new features.

Potential enhancements: complete messaging component, improve user dashboard to include a calendar of events, improve search features, and there is a lot of opportunity for animation and action to improve experience.

This would also be a good project to convert to React Native.

Overview

This is a MVP for an app to facilitate improving code skills on the go.

People check Facebook or Twitter on their phone, if only to fill a few idle minutes. Why not convert that time into mastering a language or framework!?

Say you have 10 minutes to spare waiting for a bus, between meetings, etc. You could answer a few questions, get immediate feedback, and leave with at least 1 or 2 concepts in your head that can soak in while you do other things.

Key Features

Concealed Answers: User cannot see the answers with developer tools or otherwise.

Instant Feedback: You don't have to get to the end of a quiz to see how you are doing.

Stop-And-Start: You can do as much or little as you like, close the app, and return exactly where you left off.

Links to Resources: This is a quiz, not a tutorial. Link to sites where you can learn.

Challenges

The challenges are under the hood, particularly server-side scoring with minimal latency perception on 3G. This required care to lessen computational complexity.

Since the server is pinged with each question answered, the user can stop at any time, and when reloading the app, can pick back up at the last unanswered question.

Upcoming

This is but an MVP... I want to add a lot of features, such as creating rankings/badges, utilizing user's scores and history.

I've started a companion app for daily collection of the questions in my head as I learn, troubleshoot, and debug. Later I will incorporate content and comment submission into the app, and will migrate the app to codetogo.io.
